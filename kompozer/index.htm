KompoZer

Vorwort

Ich suche immer nach Möglichkeiten auf einfache Art und Weise Texte in das HTML Format zu bringen.

Einfach HTML WYSIWIG Editoren scheint es ja nicht mehr zu geben. Zwar bieten diverse Textverarbeitungen, zum Beispiel LibreOffice oder Abiword, Funktion an, mit denen Texte im HTML Format gespeichert werden können. Es werden aber auch CSS Formatierungen mit eingebaut und die gefallen mir gar nicht. Wo man dieses Verhalten ausschalteten kann habe ich bis jetzt noch nicht gefunden.

KomboZer ist da da einen Versuch wert. Das Programm wird zwar nicht mehr weiterentwickelt aber es funktioniert noch.

Installation

Auf der Seite http://kompozer.net/download.php findet man die nötigen Pakete um KompoZer auf dem Rechner zu installieren.

In meinem Fall habe ich mich für die Version 0.8b3 (2010-02-28) entschieden. Nützlich ist auch die Sprachdatei für die automatische Rechtschreibprüfung.

Hat man sich die beiden Dateien auf den Rechner geladen geht die Installation einfach. Eigentlich braucht gar nichts installiert werden. Man muss nur das Archiv kompozer-0.8b3.de.gcc4.2-i686.tar.gz auspacken.

tar -xzvf kompozer-0.8b3.de.gcc4.2-i686.tar.gz

Nach dem Wechsel in das Verzeichnis kompozer, cd kompozer, kann das Programm auch schon gestartet, ./kompozer, werden.

Einstellung

Bei der Einstellung des Programms, Bearbeiten -> Einstellungen, habe ich noch ein paar Sachen angepasst.

Bilder

Mit base64 kann man Bilder direkt im HTML Quellcode verankern. Diese brauchen also nicht mehr mit hoch geladen werden denn die Bildinformation findet man in der HTML Datei. Um an den base64 Code ran zu kommen gibt es verschiedene Programme. Ich verwende pandoc. Der Befehl

echo '![](tux.jpg)' | pandoc --self-contained | grep "<img"

erstellt den entsprechende Code der dann einfach in die HTML Datei kopiert wird.


Fazit
