Rumex Startdateien einfach mit Symlinks

Die Rumex (g)vim Unterstützung rumex-gvim, rumex-vim und auch rumex-tilda können jetzt als symbolischer Link erstellt werden. Die erforderlichen Einstellungen werden aus der .rx/config.mk gelesen.

Die .rx/config.mk hat zwei dazu neuen Parameter bekommen.

Autor für den Weblog Eintrag
WEBLOGAUTOR = "IT-Bayer"

Bildname für das Seitenbanner
SEITENBANNER = "rxtpl/img/rumex.png"

Diese Parameter werden von den Startdateien eingebunden so dass die einzelnen Bashskripte nicht mehr angepasst werden müssen.

Es reicht ein
ln -s ~/rumex-gh-pages/.rumex/bin/rumex-gvim.sh ~/bin/rumex-gvim

um die Startdatei für Rumex zu erstellen. Verzeichnisse und Parameter werden beim Starten automatisch ermittelt und gesetzt.

Rumex kann jetzt auch mit tilda

Im Abschnitt “Homepage Änderungen, schnell und immer aktuell” findet man eine Beschreibung wie man Rumex mit tilda verheiratet um so noch schneller Änderungen auf der Rumex Seite machen zu können.

Rumex geht in die Beta Phase

Es ist so weit. Rumex geht in die Beta Phase. Zwar wurde Rumex schon auf it-bayer.de in der Alpha Phase eingesetzt, in der Beta wird jetzt noch eine Seite bzw. eine Seite und zwei Unterseiten auf Rumex umgestellt.

Wer will ist herzlich eingeladen mit zu machen.

Rumex Update Funktion wird eingebaut

Rumex bekommt eine Update Funktion. Dazu wird die Verzeichnisstruktur umgebaut. Die interne Versionsnummer für die Updatefunktion wurde auf V0.7.xxx festgelegt.

In diesem Versionsabschnitt geht es also Wild zu.

Rumex bekommt eine interne Suche

In Rumex wurden zwei JavaScripts eingebaut. Diese ermöglichen zum einen das die Seiten von Rumex durchsucht werden können. Zum anderen das die gefunden Wörter auf der jeweiligen Seite markiert werden.

Rumex bekommt eine (g)VIM Unterstützung

Da ich hauptsächlich mit (G)VIM an Rumex arbeite und hier schon ein paar .vim Schnippsel entstanden sind, wurden diese jetzt auch eingebaut. Die Erweiterung besteht zur Zeit nur aus ein paar Kurztasten und einem GVIM Menü.

Rumex’s RSS Feed hat nun auch ein Datum bekommen

Der Newsreader Lifera hatte, durch das fehlende XML TAG <pubDate></pubDate>, immer wieder Probleme mit der Anzeige der Rumex News. Rumex rss.rx0x wurde darum um das Feld | Datum: erweitert. Das passende Datum bekommt man über die Shell mit dem Befehl date -R, bzw. unter vim mit :r!date -R<CR>kJ.

Rumex bekommt eine Kommentar und Forum Funktion

Rumex wurde um eine Kommentar und einer Forum Funktion erweitert. Verwendet wird dazu das Angebot von moot.it

Es ist nun möglich jede Seite mit einem Kommentarabschritt zu versehen. Für das Forum wurde eine neue Seite forum.html erstellt die auch als Link in die Fusszeile eingebaut wurde. …weiter lesen

rumex Systemverzeichnisse werden dot Verzeichnisse

Die rumex Systemverzeichnisse wurde auf dot Verzeichnisse, nicht sichtbare Verzeichnisse, umbenannt. …weiter lesen

rumex bekommt einen Verzeichnisschutz auf System Verzeichnisse

Um die Systemverzeichnisse nicht nur durch eine Weiterleitung zu schützen wurde eine .htaccess Datei eingebaut. …weiter lesen

rumex bekommt ein Installationsskript

In rumex wurde um ein kleines Installationsskript eingebaut. …weiter lesen

rumex bekommt sein eigenes Verzeichnis

rumex bekommt sein eigenes Verzeichnis und seine eigene Datei Erweiterung. …weiter lesen

rumex bekommt jetzt auch einen sitemap.html

Nach der sitemap.xml kann rumex jetzt auch eine sitemap.html erstellen. …weiter lesen

rumex bekommt eine Weiterleitungsseite

Einbau einer Weiterleitungsfunktion per JavaScript da .htaccess von github.com nicht unterstützt wird.

rumex bekommt eine Bildbearbeitung

Eigentlich ist es gar keine richtige Bildbearbeitung sondern nur eine ImageMagick Befehlsfolge die, die Bilder im Verzeichnis bilder in verschiedene Größen bereitstellt.

Unterstützt werden zur Zeit die Größen

  • 100
  • 200
  • 400
  • 600
  • 800

rumex bekommt eine sitemap.xml

Mit einer sitemap.xml erleichtert man den Suchmaschinen das finden einzelner Seite. Man kann auch sagen sie sitemap.xml ist eine Liste der Seiten die sich auf der Webseite befinden.

Natürlich braucht rumex auch so einen Datei. Erstellt wird die sitemap mit make sitemap oder bei make all bzw. make online. .ersteckt markierte Dateien *.md?v werden natürlich nicht aufgenommen.

rumex wird auf einen nicht github.com Server installiert

rumex funktioniert auch auf einen nicht github.com Server. Eine Beschreibung wie man den Baukasten dort installiert findet man auf der Seite http://rumex.it-bayer.de.

rumex bekommt die RSS Feed Funktion

In den rumex Baukasten wurde eine RSS Feed Funktion eingebaut. Die Einträge werden dabei auch aus einem pandoc Dokument gelesen und nach rss.xml geschrieben.

Zusätzlich ist diese Datei auch als rss.html unter http://www.it-bayer.de/rumex/rss.html verfügbar.
