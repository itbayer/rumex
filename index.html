IT Bayer’s rumex

Der(Ein) HomePage Sand(Bau)kasten

github.com Version

Dies ist ein Projekt das mit pandoc's markdown und ein paar Skripten eine Internetseite erstellt und verwalten.

Gehostet werden kann diese auf github.com wo auch diese Seite zu finden ist aber auch auf Servern mit git Unterstützung oder mit einem einfachen FTP Zugang kann dieses System verwendet werden.

  • Rumex auf github (diese Seite)
  • Rumex auf einem Server mit git Unterstützung
  • Rumex auf einem Server mit FTP Zugang
  • Rumex auf einem host1free.com Server mit git Unterstützung

Der aktuelle Stand dieses Baukastens kann als ZIP Datei herunter geladen werden.

Aktuell wird dieser Baukasten auch auf der Seite http://www.it-bayer.de und auf der Seite http://www.hochfelder.de1 verwendet.

Rumex und pandoc 1.9.4.2

Am 12.10.2013 um 12:37 schrieb IT-Bayer

pandoc 1.9.4.2 kennt die Option --toc-depth noch nicht. Wer Rumex mit pandoc 1.9.4.2 verwenden will sollte diese Option in der html.mk kommentieren oder die neue Version von pandoc installieren.

Bei meiner Distribution LMDE ist leider nur die Version pandoc 1.9.4.2 enthalten. Die neue Version von pandoc muss also her. Die Installation von pandoc ist dabei nicht viel schwerer als ein Paket unter Debian zu installieren, es dauert nur ein wenig länger. Hier die einzelnen Schritte um pandoc 1.12.0.2 auf meinem T412 mit LMDE UP7 zu installieren.

sudo apt-get remove pandoc
sudo apt-get install cabal-install
sudo apt-get install alex happy
cabal update
cabal install pandoc
ln -s ~/.cabal/bin/pandoc ~/bin/pandoc
sudo ln -s /home/sb/.cabal/bin/pandoc /usr/local/bin/pandoc

Anmerkung: cabal installiert das Programm standardmäßig in Benutzerverzeichnis, Unterverzeichnis .cabal des Benutzers der die Installation gestartet hat.

Rumex Startdateien einfach mit Symlinks

Am 20.09.2013 um 07:47 schrieb IT-Bayer

Die Rumex (g)vim Unterstützung rumex-gvim, rumex-vim und auch rumex-tilda können jetzt als symbolischer Link erstellt werden. Die erforderlichen Einstellungen werden aus der .rx/config.mk gelesen.

Die .rx/config.mk hat dazu zwei neuen Parameter bekommen.

 #  Autor für den Weblog Eintrag
WEBLOGAUTOR = "IT-Bayer"


 # Bildname für das Seitenbanner
SEITENBANNER = "rxtpl/img/rumex.png"

Diese Parameter werden von den Startdateien eingebunden so dass die einzelnen Bashskripte nicht mehr angepasst werden müssen. Es reicht ein

ln -s ~/rumex-gh-pages/.rumex/bin/rumex-gvim.sh ~/bin/rumex-gvim
um die Startdatei für Rumex zu erstellen. Verzeichnisse und Parameter werden beim Starten automatisch ermittelt und gesetzt.

Rumex senden Nachrichten direkt an twitter

Am 11.09.2013 um 12:30 schrieb IT-Bayer

Dank dem Programm TTYtter for Perl ist es nun möglich Texte aus Rumex direkt an Twitter zu senden.

Eigentlich ist es nicht Rumex der die Funktion zur Verfügung stellt. In der Rumex Konfiguration wurde nur die vim Kurztaste ,rtt (RumexToTwitter) definiert. Wird eine Zeile markiert kann diese mit der Kurztaste ,rtt an TTYtter weiterleitet werden. Nach der Weiterleitung wird die Zeile entfernt. Eine andere Variante die, die Zeile stehen lässt, findet man kommentiert in der .rumex/bin/rumex.vim Datei.

Auch eine Link Verkürzung mit http://is.gd kann Rumex jetzt anbieten, Kurztasten ,rkl. Der Link für Rumex schaut dann so aus http://is.gd/az0x2a. Eigentlich ist es aber TTYtter der Rumex diese Erweiterung bietet.
