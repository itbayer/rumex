IT Bayer’s rumex (github.com Version)

Stefan Blechschmidt

2014-05-10

Rumex ist ein Projekt das mit pandoc's markdown und ein paar Skripten eine einfache Internetseite erstellt und verwalten. Diese Seiten können dann auf github.com, wo auch diese Seite zu finden ist, gehostet werden. Aber auch auf Servern mit git Unterstützung oder mit einem einfachen FTP Zugang können die Seite veröffentlicht werden.

Beispiele

  • Rumex auf github (diese Seite)
  • Rumex auf einem Server mit git Unterstützung
  • Rumex auf einem Server mit FTP Zugang
  • Rumex auf einem host1free.com Server mit git Unterstützung

Der aktuelle Stand dieses Baukastens kann als ZIP Datei holen.

Aktuell wird dieser Baukasten auch folgenden Seiten eingesetzt bzw. getestet.

  • http://www.it-bayer.de
  • http://www.byvs.de
  • http://www.hochfelder.de

Pandoc 1.12.4

Am 10.05.2014 um 19:31 schrieb IT-Bayer

pandoc 1.12.4 ist am 7. Mai 2014 erschienen. Musste natürlich gleich installiert und mit Rumex getestet werden.

Was hat sich geändert?

Der Syntax ##Weblog{.abschnitt .weblog} bzw. {.webloglink}, für den Weblog Archiv Link, funktioniert anscheinend nicht mehr, wird nicht mehr unterstützt. Die Klassen, die zur CSS Formatierung benötigt werden, wurden somit per HTML Befehl eingebaut. Außerdem wurde die Erweiterung des HTML Templates nach geflickt.

<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=yes">

Ein Update, 0.8.3.2, steht zur Verfügung.

RSS Update

Am 17.04.2014 um 17:57 schrieb IT-Bayer

Die RSS Funktion von Rumex wurde überarbeitet. Das Datum der Nachricht wird jetzt korrekt angezeigt bzw. verarbeitet. Diese Neue Funktion ist in der Version 0.8.3.0 enthalten die auch schon zum Download make update bereit steht.

Nach dem Update muss die ../.rumex/bin/rumex.vim neu eingelesen werden so dass vim das neue Datumsformat übernimmt.

:so ../.rumex/bin/rumex.vim

Achtung: Durch die Änderung der Datumsangabe in den NEWS Einträgen ist eine Überarbeitung der alten Einträge in der Datei rss.rx0x erforderlich. Diese müssen das neue Format des Zeitstempels tragen.

Das neue Datumsformat kann mit der Kurztaste ,rnn gesichtet werden, es wird eine neuer NEWS Eintrag mit dem aktuellen Datum Zeitstempel erstellt.

Der Zeitstemple / Kommentar, hinter der Überschrift, kann kopiert und für die alten Einträge verwendet werden.

Wichtig: Die alten Einträge müssen geändert werden.

Twitter und TTYtter

Am 23.03.2014 um 18:35 schrieb IT-Bayer

Twitter verlangt jetzt eine SSL Verbindung.

*** server reports: "SSL is required"

Abschnittsbeschriftung

Am 02.03.2014 um 23:43 schrieb IT-Bayer

Auf der Startseite wurden Beschriftungen für die einzelnen Abschnitte eingebaut. Die index.html setzt sich somit aus drei Teilen zusammen, die jetzt mit einer Überschrift unterteilt sind.


start.rx0s

-> Weblog

weblog.rx0s

-> Seitenübersicht

*.rx?s

Die mit -> gekennzeichneten Stellen sind neu dazu gekommen. Wird keine weblog.rx0s Datei verwendet, bzw. ist diese Datei nicht vorhanden, werden auch die Abschnittsüberschriften nicht angezeigt. Auch die Überschrift “Seitenübersicht” wird dann nicht mehr eingeblendet.

Eine neue Version 0.8.2.9 wurde erstellt und steht für das Update make update bereit.

Weblog Beschreibung

Letzte Bearbeitung: 2014-04-21

Diese kleine Erweiterung soll eine Art Blog innerhalb Rumex ermöglichen. Besser gesagt ist diese Erweiterung eher ein Weblog Funktion.

Der Unterschied zu den eigentlichen Rumex Seiten ist nicht sehr groß. Folgt nach einer h1 Überschrift ein div Tag mit der weblog Klasse wird der Text innerhalb dieses div Tags ausgeblendet und mit dem Schriftzug ... weiter lesen ersetzt. Die Überschrift und der Text ... weiter lesen bekommen eine Link Eigenschaft die den ausgeblendeten Text bei Anforderung einblendet.

Diese Funktion kann auch in den normalen Seiten verwendet werden.
